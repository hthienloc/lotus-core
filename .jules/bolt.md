## 2024-05-19 - [Performance Optimization: String utilities and Log buffer]
**Learning:** Utilizing `std::string_view` over `const std::string&` in combination with `res.reserve()` inside highly-called string utility functions eliminates unnecessary and expensive heap allocations without breaking existing caller logic. Additionally, replacing `std::vector::erase(begin())` with a fixed-size ring buffer implementation prevents O(N) operations in trace logging. Standard Library includes must always happen at the global scope, not inside namespaces.
**Action:** Default to `std::string_view` for string read access and evaluate `std::vector` modifications that act as queues for replacement with $O(1)$ ring buffer abstractions. Make sure header includes do not appear inside namespace scopes.
